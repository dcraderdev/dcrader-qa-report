## Top 5 Most Urgent Fixes

1. **CRITICAL — `https://www.dcrader.dev/work/socialstakes` returns HTTP 404**
   The `/work/socialstakes` case study page is completely missing. This is a portfolio piece that should exist; any prospective client clicking through gets a 404.

2. **CRITICAL — All `/v1`–`/v5` sub-routes are 404 across every Vercel template hub (50 broken URLs)**
   Every `dcrader-template-*.vercel.app/v1` through `/v5` URL returns 404. The index pages (`/`) load correctly but none of the individual variant routes are deployed. Affected: restaurants, trades, realestate, photographers, pets, dentists, chiropractors, trainers, salons, landscape (10 hubs × 5 variants = 50 broken URLs). Same issue on GitHub Pages for auto and tattoo collections (10 more 404s).

3. **CRITICAL — `https://socialstakes.onrender.com` returns HTTP 503 (service unavailable)**
   The live SocialStakes app is down entirely. The onrender.com free tier may have been suspended or crashed.

4. **HIGH — `https://www.dcrader.dev` homepage has 9 broken images + unresolved loading skeleton**
   Nine portfolio/work images with paths like `/_astro/01-article.D31JBnK-_1fCV4s.jpg` return 404 — likely a broken Astro build artifact or missing static files. A loading skeleton also remains visible after page load, indicating a JavaScript hydration or async-fetch failure.

5. **HIGH — All five Trades variant sites (`dcrader-v1` through `dcrader-v5`) have 6 broken images each using `picsum.photos` seeded URLs**
   `picsum.photos/seed/<name>/800/500` URLs are failing across all five trades templates. The picsum.photos `/seed/` API endpoint appears to be returning errors for these specific slugs, leaving every before/after comparison image broken. All five sites are affected (30 broken images total).
